Builder

High-quality DIY projects with great assembly instructions.

GitHub, Gitter

Builder is a set of tools to help creators organize do-it-yourself electronics, software, and product design projects into complete presentations and kits.

The concept at the heart of Builder is interactive self-assembly (or IASM). I came up with this idea in late 2015 to refer to a style of working with automated systems (potentially learning systems), generally computers, that involves a person curating the completion of work by orchestrating automated and semi-automated (or intelligent) processes that require some amount of human interaction, usually one individual. Interactive self-assembly is related to the concepts of intelligence amplification, and self-assembly (specifically self-assembling machines).

Builder leverages this concept in its core mechanism that generates personalized step-by-step assembly instructions to facilitate the design and prototyping of interactive systems (with working components, in the case where Builder hardware is also used). For electronics, this eliminates the need to reference documentation for hardware, choose pins to use, and figure out how to connect components to a controller chip (e.g., Builder boards, Raspberry Pi, Arduino). I decided to do this to take a small step toward making it possible for more individuals and communities to participate directly in technology innovation. This is also a problem I encounter every time I work on electronics projects.

Builder is specially tailored to projects hosted on GitHub. For users to get the maximum benefit of Builder, they should use it with all GitHub projects that they want to share with the idea that others will replicate the design. Once another user checks out the repository, they can use the generated results to assemble the project. Then, they can modify the project, use Builder to update the instructions and push the changes to their own repository. The effect is that sharing electronics and software projects is a better experience and the productivity of users is accelerated.

Example Usage (CLI)

To generate instructions to connect an IR rangefinder and servo to the Raspberry Pi 3, users would execute the following command:

builder connect --model raspberry-pi-3 --model machineee/generic-servo --model ir-rangefinder --model ir-rangefinder
